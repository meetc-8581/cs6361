1. Introduction

 

1.1 Project overview

This is a project plan to be used to implement the smart-phone application named Theia which is designed to help blind people to navigate indoors of the building, across the building, like going from one part of the building to another part of the building using Theia which gives instruction to the user like take a left, go straight for 50 steps and all.

This application has multiple functionalities like emergency calling to a friend/family member, getting navigated from one part of the building to another part of the building using accessibilities. It helps the user to avoid collisions using certain hardware, and helps users to get the most efficient path. This application also have setup functionalities, that another person can help the main user to configure system.

2. Project organization

2.1 Process model

We are planning to utilize the waterfall model during the development process of the project. We are currently in the requirements gathering phase. Later on, we will be doing the system design process where we will decide the hardware and software specifications of the project. After this step, we will review the design and do refinement if needed. The Implementation will be started after this phase. We plan to gather the requirements and skills of all the team members and based upon that will use the platform for the development and SDK. This will be followed by testing and then deployment.

3.2 Assumptions, dependencies, and constraints

        We have made the following assumptions:

  • The user will be only using the app for navigating indoors. No additional hardware will be available to the user.
  • Some additional users will be available to the main user for the initial configuration of the app and account.
  • The user will be ‘Tech Literate’ and will be able to make use of Tech devices with a certain level of fluency.

4. Technical process

4.1 Methods, tools, and techniques

We will implement this application using React Native. The main reason for selecting React-Native to develop this application is that this is a cross platform framework using javascript. So, we can use the same code to run on a different platform like AndroidOS and iOS. In our team, most of the team members are comfortable with javascript so that’s why we have selected React-Native. We are also using google docs to create a document that is shared. So, everyone gets up to date progress and also uses Microsoft teams for communication. Draw.io chart tool will be used to do the UML modeling.
